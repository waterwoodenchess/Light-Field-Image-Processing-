II.A
光场其实就是光线形成的一种场，他是可以被描述的，我们把描述光场的函数称作全光函数。全光函数理论上来讲应该是一个7维的函数，包括：光线位置(x,y,z)、光线角度(θ,Φ)以及光线的波长λ和时间t，所以光场函数其实就是L(x,y,z,θ,Φ,λ,t)。但是在计算机运行的过程中，我们发现一个7维的函数是很难被计算的，所以可以考虑将函数降维。

首先我们可以假设收集到的光线波长也就是λ是恒定不变的，并且每一个λ都有自己的收集通道；在拍摄的过程中时间t可以通过视频中的不同时间帧记录光场，于是波长λ和时间t就被降维，光函数从7维降至5维。
但是 Levoy 和 Hanrahan以及 Gortler 等人发现5维的函数在表达过程中还是有一些冗余，于是他们提出了一个假设，即假设光线在自由空间中进行测量，这个时候光线呈直线传播，所以可以降到4维。他们提出了，光线可以表示成两个平面内的任意一条直线，设第一个平面的坐标系是(u,v)，第二个平面的坐标系是(s,t)，所以光线的坐标就变成了L(u,v,s,t)。光线从第一个平面某一点摄入，于是我们可以确定此光线的第一个平面点(u1,v1)；然后光线再穿过第二个平面，我们就可以确定光线的第二个平面点，将这四个点组合在一起，我们就得到了这个光线的唯一函数L(u1,v1,s1,t1)。所以，我们之后都是用L(u,v,s,t)进行光线参数化的。

II.B
在L(u,v,s,t)中，我们考虑将st平面想象成一个相机阵列，而uv平面就是这个相机阵列的共焦平面。物体发出的光从uv平面出发抵达st平面，所以st平面的采样数量代表了观察到的一个物体的视角数量，我们又称它为角度维/视角维；uv平面的采样数量则代表了st平面所拍到的每张图像的分辨率（uv的采样数量越大，代表着st平面所拿到的光线数量就越多，代表着信息越大，所以分辨率高），我们又称uv平面维空间维。

在uvst平面中，我们可以固定st，改变uv的位置，所以我们得到的是st平面上某一个视角所得到的一张2D图像，我们将这个2D图称为子孔径图像/针孔视图sub-aperture image/pinhole view；当我们反过来固定uv改变st的时候，我们得到的是不同视角下穿过某一个固定位置的光线向不同方向不同位置的辐射值分布，也就是说在这个时候我们看到的是同一个点不同方向的光线，我们把这个称为光场子视图light field subview；当我们固定了v和t，也就是固定了某一排相机和某一行光线，改变s和u，即改变这一排相机的视角和这一行光线的横向位置，并且将s的光线按照s的顺序堆叠起来，我们就得到了EPI。那么为什么EPI我们看到的是一条斜直线呢？我们可以想一下，对于一条光点而言，不同的视角看到这个点的像素会不一样，比如在s=1的时候这个光点在u=100，s=2的时候这个光点在u=103，s=3的时候这个光点在u=107，s=4的时候这个光点在u=108......这样下来EPI自然就是一条斜直线，所以EPI 里的斜线 = 同一个真实物体点在不同视角中的位置轨迹。<img width="348" height="240" alt="image" src="https://github.com/user-attachments/assets/453503f4-d96b-4be3-8f28-f2dd271cd2d9" />

假象我们可以保持右边的相机的光场不变，然后向左平移，使得蓝色的光线在成像面上重合，那么最终蓝色方块在两个相机成像的照片里位置就会完全相同，这其实就等效于把原始位置成的像向左移动了一段距离，然后和左边相机成的图像叠加，那么就会发现蓝色方块重合了。类似的，如右图所示，如果把右边相机成的图像向左移动一大段距离，那么更远的绿色三角图像就重合了，要想是不同位置的物体重合就要对应不同的移动距离，而这个距离实际上是和物体到镜头的距离相关的，通过移动距离和相机采样点之间距离的比值就可以轻易求出。所以EPI为什么重要？因为我们可以通过EPI得到物体的深度关系，对于近处物体：视角一变，它在图像里移动很多；远处物体：视角一变，它在图像里移动很少。这就是视差。所以在 EPI 里：近处物体形成的线，位置变化快；远处物体形成的线，位置变化慢。  而EPI斜线斜率和深度的公式在论文里面也有所体现：Δu = (f/Z)· Δs，其中相机视角移动 Δs、图像位置移动 Δu，二者比例和深度 Z 有关，f是uv和st平面之间的距离。Z 越小，物体越近：f / Z 越大、Δu 越大，也就是同样移动相机，近处物体在图像里跑得更厉害；Z 越大，物体越远：f / Z 越小，Δu 越小，也就是远处物体几乎不怎么动。所以 EPI 的核心用途是：看线的倾斜程度，就能推断深度。不用纠结图里线到底是往左斜还是往右斜，也不用纠结“更陡”还是“更平”，因为这取决于图中把 u 放横轴还是竖轴，只要抓住本质：EPI 线条的方向/斜率 = 视差，视差大小 = 深度信息。

II.C
在上文中我们已经知道了光线的表达形式以及如何采集。在这一部分中我们讨论的是如何从已经有的视角图中渲染出一个全新视角的新图。
想象我们已经有了一圈相机1、2、3、4、5......，每个相机都能拍到一个自己视角的2D图像。但是我现在想要相机3.5处的视角图，我应该怎么做？这就叫做光线渲染Light Field Rendering。在光场中，我们的渲染很直接，就是我现在需要一个新的光线，我可以从已经有的光线中挑选和这个视角相近的光线进行插值，所以步骤就两步：第一步在已经有的光线中重采样，然后再插值。所以论文里面说当相机的分布足够密集的时候，我们就可以通过光线的重新插值来合成一个新视角图像，如果这个视角需要的光线和已经有的光线是同一条，那么我们可以直接拿来用，但一般情况下我们都需要插值来重新估计光线，而合成的一般大前提是光线在没有遮挡、没有吸收等条件下亮度颜色均不发生改变。在合成的时候，我们通常需要16条光线进行合成，u方向2条、v方向2条、s方向2条、t方向2条，2×2×2×2等于16条。

如果采样不够密，那么在插值的过程中会出现伪影的情况，因为如果采样不够密集，那么在已有的光场中不同视角所拍摄到的同一个点的位置会有极大的差别，因此在插值、合成之后会出现一个点有重影的情况，这种现象就是伪影。所以论文里面提到，为了防止伪影的出现，我们必须要保证采样的视角差小于一个像素值。也就是相机越密，采样点越多，越容易插值合成清晰的照片；相机越稀疏，采样点越少，越不容易插值合成。而且EPI也可以给插值合成起到一定的辅助作用，主要体现在深度可以帮助采样数量减少。为什么？因为如果知道了深度，那么我们就可以知道在不同视角下同一个点的移动情况，就能够明白不同视角下像素的对应情况，因此就能减少光线的采样点；如果不知道深度关系，我们就不知道不同视角下像素的对应关系，就需要非常密集的光线采样来确保合成图片的准确性。

II.D
在已经知道了EPI之后，我们通常会考虑使用傅里叶变换来研究EPI的频域特性。

如上面这张图所示，(a)是EPI的一条斜线，(b)是将EPI进行傅里叶变换之后的频域特性线，(c)和(d)之后我们讲。
先来看a和b。对一条空间域的线进行傅里叶变换，我们得到的频谱域的线的斜率通常是和空间域的线的斜率呈垂直关系，所以b的斜率和a的斜率其实是垂直关系。我们再来看b，为什么b的线不是和a一样是一条单纯的线，而是一整个嵌套的椭圆环结构？因为按照理论上来讲，EPI的空间域线是一条无限长、无限细、完全直、亮度恒定的一条线，那么相应的他的频域线也是一条直线。但是实际的EPI线是有厚度、有限的一种局部结构，所以他的频谱域线会是细长的、倾斜的、带厚度的频谱区域，这种频谱线类似于高斯形状，是一种等值线，所以在频谱画的时候我们能看到是按照某一个方向展开的嵌套的椭圆曲线，其实就是等值线。

我们再来看c和d，首先不要管蓝色和红色的框子，单纯的看密集的频域图。首先为什么会有这种密集的频谱图？我们在进行采样的时候，通常进行的是离散点采样，而傅里叶分析里有一个非常重要的结论：在空间域采样，会导致频域出现周期性复制。也就是说原始的频域图只有一份，一旦我们对光场进行离散采样，他的频域就会出现很多份对原来频域图的拷贝。所以这就是为什么c和d会出现这么密集的频域图的原因。

我们再来看c和d的红蓝框。首先我们要明确一个观点，那就是，采样太密，这些复制品会离得比较远；采样稀疏，这些复制品会离得比较近，甚至重叠，这就是我们常说的的“混叠现象”，（傅里叶理论里有一个结论：空间域采样间隔是 Δx，频域复制间隔就是 1 / Δx，所以空间域采样越密，间隔越小，那么他的频谱图就会越稀疏，反之越密集），混叠现象会导致图像渲染的时候出现鬼影，因为混叠现象导致的频谱重叠会使得重建原始信号时无法将信号剥离开，因此会导致原始信号和复制信号的重复，因此出现伪影现象。在图c里面，蓝色的框代表的是普通的滤波器，我们可以看到，蓝色的滤波器原本是想要把最中间的频谱图框住，但是会把邻近频谱复制品的一部分也框进来。所以普通的滤波器在这个时候根本不行。所以就有了图d的红色框，也就是沿着频谱方向做一个更合适的滤波器。因为这个 EPI 只有一条斜线，频谱结构比较简单，所以可以用一个倾斜的滤波区域把正确部分保留下来，同时避开旁边复制品。这叫：sheared reconstruction filter剪切/倾斜重建滤波器。它的设计直觉就是频谱是怎么斜的，我这个滤波器就是怎么斜着取。

当然，上面这些都只是谈论的是简单情况。实际情况中要比这复杂很多。

III.A、B
这两章主要就是讲怎么获得光线的，主要是硬件方面的知识，主要分为多传感器捕获（好几个视角拍照）以及时序捕获（用一个相机在不同的时间拍照）。这里不过多赘述。

III.C
这一章所讲的内容就是：如何用一个相机、一次曝光把4维的光场数据放到只能收集2维数据的传感器里面。这里面论文提到了两种方法，一种是讲把不同视角放在传感器图像不同空间位置上面，也就是空间复用；一种是把不同视角放在传感器图像不同频率里面，也就是频率复用。

首先我们来讲讲空间复用。空间复用就是在传感器上面生成了很多张交错排列的小图像。比如论文里面提出的Lytro、Raytrix、plenoptic camera 1.0，这几个人属于比较典型的微透镜阵列方法论的。他们的思路是这样的：把相机设计成为主镜头 → 微透镜阵列 → 传感器（从外到内）这种形式（每个小微透镜下面是一小块传感器像素，一小块传感器像素数量可能有100、400......个）。这种设计是这样考虑的：不同的微透镜所代表的是不同的视角，微透镜下面的传感器像素又收集不同方向的光线，比如左边的像素收集左边的光线，中间的收集中间的光线，右边的收集右边的光线，他是将不同视角直接放置在不同的空间上面，所以叫做空间复用。但是这个方法有一个问题是空间分辨率和角度分辨率之间会抢资源，这就是论文里面提到的trade-off，为什么呢？我们假设传感器总共有1000×1000=1000000个像素，如果我们把这些像素给每一个透镜下面放100个，那么我们可以得到大约1万个透镜，如果把每个透镜下面的像素值提高，那么我们的透镜就会减少。所以就是说：空间分辨率（透镜数量）和角度分辨率（透镜下面的像素数量）之间有一个折衷trade-off。当然会有人问为什么不把传感器的总像素值提高？这样做理论是可行的，但是第一，传感器尺寸不能无限大，相机的镜头只能把一定尺寸的图像投射到传感器上面；第二，我们不能给传感器塞更多的像素，让每个像素变小，像素变小进光量会变小、动态表现会变差、暗光范围会变小；第三就是每个像素的大小受限于光学特性，太小光学特性会变差。

其次就是频率复用。他的核心思路在于将不同的2维图像通过不同的编码方式放在图像不同的频率带里面。常用方法是modulation mask / attenuation mask 调制掩膜 / 衰减掩膜。也就是在光路上放一个调制好的掩膜，不同方向来的光线通过不同的掩膜图案之后会落在传感器不同的傅里叶频域上，然后通过解码或者别的什么操作（可能是反操作，比如它是根据线性关系把光线进行编码的，那我在解出原光线的时候进行反操作就好了）把光线从傅里叶频域里面解出来。这种操作的缺点在于他的每一帧曝光时间都比较长，然后通光量也会减小。

IV.
这一章提出了两个问题：如何使得空间分辨率更加清楚以及如何使得角度分辨率更加清楚？也即如何让每个视角图变得更加清楚以及如何拥有更多角度。

A.提升空间分辨率（空间超分辨率）
首先为什么要用空间超分？因为单视角的空间分辨率太低了。传统的方法是对每一个单视角图像用超分方法放大。什么是单视角超分方法？就是给算法一张单视角低分辨率的图，让算法生成一张高分辨率图。普通的算法是用插值的方法进行补点，但是这样子只能提升图像的平滑度。现代的方法更多的会使用深度学习，会学习低分辨率图像长什么样、高清细节通常应该怎么补、边缘、纹理、人脸、文字可能怎么恢复，所以它不仅插值，还会“猜”高频细节。但问题也在这里：它只能根据一张图猜细节。这种效果其实并不好，并且也没有充分利用不同视角下图像的信息。所以就有了下面的空间超分。
空间超分的核心思路是利用单张图像的超分方法，也就是将每一个子视角图像都变得清楚，换句话说就是把很多张低分辨率的图像合成成一张高分辨率的图像。方法在于由于不同视角的图像之间差异经常是亚像素（0.1个像素点、0.3个像素点这样子，就表示不是一个完整像素），所以我们可以选定一个视角（比如中心视角）然后通过不同的视差来计算每一个像素在目标视角的落点在哪里，这样子可以补充目标视角的高频细节，这样子我们就可以通过融合很多张不同视角的图片得到一张高分辨率的目标视角图像。

B.提升角度分辨率（角度超分辨率）
这一讲论文的核心思路是通过已知的角度信息来合成新视角的图像，这样子可以提升角度信息。论文提出来了两种方法，一种是基于深度/视差的角度超分，一种是基于光场结构的角度超分。首先第一种基于深度的角度超分，如果我们知道深度，我们就可以知道不同视角要移动多少像素才能到目标视角，所以通过深度我们可以把很多张低分辨率的图像通过像素移动合成一张新视角的图像。这里面有一个很重要的问题空洞。意思就是有些地方已知视角没有，但是目标视角会有，但是由于移动以及合成的时候这个地方没有像素来源，所以目标视角这个地方会形成一个空洞，所以我们还要处理目标视角的空洞，当然我们还要处理深度错误、边缘错位、非朗伯反射这些错误。

其次，我们来讨论第二种方法，也就是基于光场本身结构来进行合成的思路。这个方法主要依靠学习EPI的结构，因为我们已经知道的EPI，需要合成的是空缺的EPI，如果新的EPI符合结构，那么这个新视角图会是一个很不错的图。所以我们只需要学习或者恢复EPI的线结构就好了，传统方法可以是稀疏表示、低秩模型......，现在的方法可以是CNN、Transformer......他的本质还是：根据已有视角，补出中间视角，使 EPI 结构连续合理。

前面 III 采集说过：空间分辨率和角度分辨率会互相抢资源。所以 IV 超分辨率就是来补救这个问题。如果采集时空间分辨率低：做 空间超分；如果采集时角度分辨率低：做 角度超分。有些方法还会同时做：空间-角度超分，也就是既让图更清楚，又让视角更多。一句话总结光场超分就是在补光场的采样密度。

C.提升时间分辨率（时间超分辨率）
提升时间分辨率换句话说就是提升帧率。假设我们有t=0的帧0、t=1的帧1，我们需要更好的帧率，那么我们就需要进行插帧。普通的插帧一般都是通过前后两帧的画面来猜测中间这一帧的画面，但是光场插帧必须要考虑新帧的所有视角关系，不能出现插入的新帧正面看好着结果左面看出现遮挡或者视角关系不对的情况出现。所以光场插帧通常都是依靠前后两帧的光线、视角、空间之间的关系将新帧进行插入操作。

D.光谱重建
我们平常的相机采集的都是R、G、B信号，但是现实生活的光有很多波长，光谱重建的想法是通过采集一小部分RGB信号，然后将光谱信息进行重建。为什么要重建光谱，因为光谱的信息含量比RGB信息含量大，RGB只能告诉你这个地方是红色、绿色还是别的什么颜色，但是光谱可以告诉你这个地方的辐射波长和反射是多少，这在很多领域比如材料、地质勘探等等领域都很有用。并且有很多情况是有相同的RGB色值却又完全不同的光谱波长，所以重建光谱比恢复RGB更深入、也更有意义。在RGB恢复成光谱过程中，通常很大程度上依赖先验知识和训练数据才能把数据恢复好。

（非郎伯区域：从不同方向看，亮度/颜色会变化的区域。朗伯表面是理想漫反射表面，比如比较接近：哑光墙面、白纸、粗糙木头、粉笔表面，它的特点是：不管你从哪个方向看，它看起来颜色/亮度基本一样。但是非郎伯区域就是完全相反，比如镜子、金属、有高光的玻璃杯、水面等等，以水面为例，我们从左边看高光可能在右边，但是从右边看高光可能在左边，这就是在不同视角下亮度不一致。在很多算法里面，大家都默认图像里面的物体是郎伯区域，这样方便进行深度估计、像素匹配以及EPI的合成，但是非郎伯区域的出现会导致不同视角看到的像素值不同，很容易导致像素合成的失败以及新图合成的伪影。）

V.光场图像中的深度估计
这一讲主要讲的是如何估计光场图像里面的深度。请注意，这里面说的深度是指视差（你可以大致理解为这个物体或者这个点距离镜头的距离，还记得我们之前说过的EPI吗？差不多是那个，EPI的斜率代表了视差，斜率大=视差大，反之视差小）。视差和深度是反比关系，视差大，深度也就小，物体离得近（不同视角下物体移动的像素也就多）。
所以论文提到了深度估计总共分为2步：第一步就是对深度做一个初步的估计，把大致的深度关系尽量得到；第二步就是对第一步初步得到的深度估计进行修正，使其更加符合边界。也就是论文的A和B部分。

A.深度初步估计
论文在这一部分提出了多种方法，一种是基于匹配的方法，这是一种很简单也很直观的思路，就是看目标点在不同视角下移动了多少像素。比如我们要估计A点的深度，这个A点在左边视角处于100这个像素值，在右边视角这个A点处于104这个像素值，从左视角到右视角这个点移动了4个像素值，所以这个点的视差就是4；而另外一个点B用相同的方法测出来视差是10，所以A点比B点远。这种匹配的方法基本流程是：1. 选一个参考视角，比如中心视角；2. 对参考图像中的每个像素，在其他视角中找对应点；3. 找到移动量，也就是视差；4. 把视差转换成深度。这种方法较为简单，但是他在低纹理区域并不好匹配、遮挡区域不好找到匹配点等等问题。

另外一种方法是进行重聚焦，意思就是我们可以估计一个深度，然后把不同的视角按照这个预估深度进行对齐，如果对齐之后清晰了，那么这个深度就是对的，如果对齐之后不清晰是模糊的，那么就说明这个深度是不对的。所以我们可以尝试很多深度，看到底哪个深度能让视角对齐清晰。进行重聚焦的时候文章提到了两种线索：一种是通过不同视角对齐看颜色进行深度对齐，一种是看哪个深度在重聚焦的时候更清楚。这两种可以一起进行，但是他们二者之间是有区别的：颜色对齐：就是在目标点位对齐之后从每一个视角来看颜色是否一致，如果不一致那就是深度错误；清晰度对齐：就是在目标点位对齐之后看目标点位的边缘是否清晰光滑，如果不清晰那就是深度错误。会有人问：不是只要对齐之后清晰就可以了吗，为什么还要颜色进行对齐？在不同的视角下，同一个目标点位的像素其实是不一样的，同样一个点位在中心视角可能是100，左边视角可能是96，右边视角可能是在104，我们进行深度估计后进行对齐操作，如果深度估计正确，对齐之后我们就知道每个视角要平移多少使得这个点位边缘更加平滑，而且从不同的视角来看这个点位颜色一致；如果深度估计错误，可能按照这个深度对齐之后这个点位无法对齐，边缘模糊，而且从不同视角看到这个点位的颜色并不一致。这么说可能并不清楚，但是对于比如条纹衣服或者栅栏这种情况，如果我们光使用清晰度对齐很有可能把条纹对错但是边缘依旧清晰，这个时候颜色对齐就变得很重要，能够判断不同视角下这个点位是不是同一个点，避免不同的点因为清晰度对齐而被对在一起的情况发生。

第三种方法是通过EPI斜线进行分析。他是这样做的：首先我们在图像中在水平或者垂直方向构建EPI；然后我们通过这个EPI来分析局部的结构方向，怎么分析呢，就是从EPI的线方向来看这个这里的方向/边缘向哪个方向延伸，如果一个EPI线向右下方延伸，那么这个方向的几何体是向着右下走的。当然了因为我们有眼睛，所以我们当然知道这条线的方向走向，但是计算机并不清楚他的方向，所以我们需要一些工具来计算这个方向，常用的工具是方向张量。核心思路是计算最小变化方向，对于一条线来讲，最大的变化方向是垂直这条线，而最小的变换方向则是沿着这条线。所以我们通过计算不同方向的梯度来得到最大和最小的变化方向。只要知道了最小的变化方向，那么我们就知道这条线的方向。这种方法有一个好处就是在EPI线被截断的时候可以知道每个小局部的方向，当然它也会在低纹理或者交叉线区域出错。

第四种方法就是用深度学习来学习，比如CNN和Transformer。传统算法是依靠人来进行手写算法，比如各种规则。但是深度学习是依靠神经网路来学习不同视角之间像素怎么移动、对应什么深度、EPI 里不同斜率对应什么视差、遮挡边界处应该怎么处理、低纹理区域该参考什么上下文，然后得到一个自己学习的深度图，将这个深度图和真实的深度图进行比较计算误差函数，然后反向传播给这个网络，调整参数，学得多了网络就会自己判断各种复杂情况的处理方式。

有的人可能会有疑问说既然我们之前都说了EPI的斜率就是视差，为什么我们还要搞这么复杂？为什么不能直接算EPI的斜率呢？没错，按照理想的情况来看确实是这样的，EPI是一条完整的斜线。但是现实情况是每个物体之间可能会有遮挡、可能没有纹理等等各种复杂情况，所以EPI也会变得很复杂，比如线被截断、很淡、多条线交叉、一个位置有两条线、这个地方的EPI是一些点没有线等等等等，所以我们需要知道哪些线是真实的、对于没有线的地方我们要知道斜率是多少、滤除噪声等等等等，所以不能只依靠EPI来进行估计。

B.深度修复
在进行了第一步的初步深度估计之后，我们通常会进行深度的修复，这是因为第一步通常会出现很多问题，比如噪声、空洞、边界错位等等等等，所以我们要在这一步将原先没有搞好的图进行修复，使得深度图更加平滑，也保留物体的边界。当然了只是平滑物体内部的深度，物体边界的深度是可以突然变化的。文章提出了好几张方法：1. Edge-aware refinement/边缘感知优化，也就是用中心视角图像的边缘指导深度图，因为图像颜色边缘处，可能是深度边界，而图像内部平滑处，深度也应该平滑，所以颜色相似的邻近像素，深度也相似、颜色差异很大的边界，不强行平滑深度；2. Occlusion handling/遮挡处理：遮挡是光场深度估计里很麻烦的点，比如前景物体挡住背景，从左视角看，某个背景区域被挡住；从右视角看，它可能露出来，在 EPI 里，遮挡会表现为：不同斜率的线相交或截断，这时候一个像素附近可能同时有前景线和背景线，初始估计很容易选错，所以Depth refinement 会尝试识别：哪些区域是遮挡边界、哪里应该信前景、哪里应该信背景......这样可以减少边界毛刺和错误深度。3. Confidence map/置信度图：有些方法会同时估计一个置信度图，意思是：这个像素的初始深度估计可信不可信，比如：纹理丰富、匹配一致->高置信度、低纹理、遮挡、反光 -> 低置信度，所以在修正的时候高置信度区域多保留、低置信度区域更多依赖邻域传播和图像引导，这样比盲目平滑更合理；4也就是最后一种，是全局优化，意思就是在全局的角度来确定深度而非单纯只是一个点来确定深度，他的核心思路是要求每个像素的深度既要符合观测、也要和周围的像素协调，很多方法会用一个能量函数来估计全局的优化，就是综合考虑目前这张图与观测是否一致、本身深度是否平滑，当然了，我们不能只有平滑估计否则会将物体的边界丢失掉，举个例子：假设一行像素的初始深度是：2.0  2.1  8.5  2.0  2.1
这些像素都在苹果内部，颜色都差不多是红色，观测值可能说：第 3 个像素我不太确定，depth=8.5 也许是错的；光滑说：旁边都是 2 左右，中间突然 8.5 很奇怪，优化后可能变成：2.0  2.1  2.1  2.0  2.1这就是修掉孤立错误点；但如果一行像素是：苹果 苹果 苹果 | 背景 背景，深度是：2.0  2.1  2.0 | 7.0 7.1，颜色边界很明显，光滑会在边界处放松，不强行把 2 和 7 拉近。

VI.光场评价&压缩
这一讲论文主要讲的是如何评价一个光场图像的质量并且把他进行压缩。

A.评价光场质量
光场质量的评价比普通图像的评价要复杂一些，因为普通图像的评价一般都是看是否模糊、有没有噪声、有没有损坏，但是光场的评价还要看各个视角之间是否一致、EPI是否平滑、不同视角切换的时候是否平滑等等。论文给出了三种评价类型：第一种，空间-角度分辨率，这个就是看光场的这两个分辨率是否足够，这个主要是看相机对高频细节的保存能力，因为如果这个能力越差，拍出来的边缘马赛克会越清楚，这个能力越好，拍出来边缘就越锋利越清楚；第二个就是空间图像质量，这个和普通的图像评价一样，是看PSNR和SSIM，PSNR 越高，通常说明误差越小，SSIM 越接近1，结构越像；第三个就是看角度的一致性，这个是光场评估的特点，主要看不同视角切换的时候是否连贯，比如我们新插了一个视角图，看着还可以，但是在进行视角切换的时候会发现新视角图相比于其他的图会有跳变的情况出现，这就是角度一致性差的体现，所以评估光场图像质量不能只看单张图像，还要看各个视角，要看连起来是不是像一个完整的光场。

B.光场图像压缩
文章把光场图像压缩分成了两个部分：有损压缩和无损压缩。为什么要压缩，因为这样能够方便传输、存储以及显示。

第一个，有损压缩
有损压缩，顾名思义，就是会产生损伤的压缩，这种压缩会损失一部分光场的信息，但是他的压缩率比较高。论文提出了三种有损压缩的方法：变换编码、预测编码以及伪序列编码。首先我们讲变换编码。变换编码的思路和普通图像压缩的思路有一点类似，普通图像的压缩是计算图像的频率系数、然后丢掉不重要的高频最终进行编码压缩，但是对于光场图像来讲，由于其具有四维结构所以不能把光场图像单纯的当作普通的图像进行压缩。所以有人考虑利用光场图像的空间以及视角的相关性来进行压缩，基本思路就是因为光场图像每个视角都比较相似，所以把他们放在一起进行压缩，把相似的部分存放在系数里面，剩下不相似的系数可以不存或者少存，这就会导致不相似部分信息的损失。这种方法还有一个问题就在于如果相邻的视角视差太大，就是太不相似了，物体在没有对齐的情况下进行压缩，就会导致高频变多，压缩效率下降，意思就是压缩损失的部分过多。所以有一些方法在进行变换编码之前会进行视差对齐，就是先把相似的部分对齐然后进行压缩编码。

其次就是预测编码。这种方法的核心思路和视频编码的思路很像，视频编码的思路是只存关键帧，剩下的不重要帧依靠已经存了的关键帧进行预测。那么光场图像的思路就是我们只选取一部分关键视角完整存储，剩下的视角就依靠已经存储的关键完整视角进行预测，因为每一个视角之间都很接近，所以需要预测的视角其实等于已经存储的完整关键帧+两个视角之间的视差，所以我们只需要存储两个：一个就是完整的关键帧，一个就是不同视角和关键帧之间的视差值。

最后就是伪序列编码。这种方法的核心是把图像变成视频，然后用视频压缩器进行压缩。步骤是：把一张图像按照一定的顺序排列成一串，然后把这一串当成视频帧送进视频压缩器进行压缩。常见的排列顺序是“之”字形和螺旋形。如下图所示。<img width="630" height="288" alt="image" src="https://github.com/user-attachments/assets/09fb0fb4-6231-4e46-a60f-15e95ba3c65c" /> 我们为什么说路径很重要呢？因为如果路径选择的好，帧和帧之间会是相邻的视角，这样子差别比较小，容易预测压缩效率也会变高。论文说螺旋形比之字形更好，因为它从中心往外平滑变化，视角变化更连续。

第二个，无损压缩
无损压缩，和有损压缩相反，就是不会产生数据损伤的压缩，要求压缩之后解码不产生任何信息的损失，但是他的压缩率比较低。论文提到了两种思路，一种是用量化深度图将已经分割的关键视角图传播到其他视角，然后再用已经编码了的关键视角图去预测其他视角的图。什么意思？就是我们的核心思路是通过已经有了的视角去预测别的视角的图，但是不同深度的物体移动的位移不同，这就不能用同一个唯一去进行计算，所以我们把关键的视角图分割成好几个部分，这几个部分需要内部纹理、颜色、形状等等等等相似，因为这样就能用相似的视差进行预测。然后我们使用量化的深度图估计关键视角图的每一个部分在别的视角里面需要移动多少。什么是量化深度图？就是把深度图的深度进行量化，把所有深度分成好几个等级，比如近处的物体深度大概需要挪lv1的深度，更远一点的深度大概需要lv2，再远一点的在lv3......这就叫把分割的部分用量化深度图传播到其他视角。再之后我们就可以用已经编好码的关键视角和误差来预测其他的视角图。这是其中一种思路。另外一种思路是对传感器拍到的原始光场图直接进行压缩。由于相机在拍摄的过程当中并不是一个像素全部记录RGB，而是不同的像素记录不同的颜色，所以我们在压缩的过程中将RGB分离开来，然后我们不要直接存当前像素值；先用周围已知像素预测当前像素；再只存预测误差。比如当前像素真实值是：103，根据左边、上边像素预测它是：101，那只存：误差 = 103 - 101 = 2，误差通常比原始像素更小、更集中在 0 附近，所以更容易压缩，因为真实值=预测值+误差，所以这就是无损的。并且我们在压缩的过程中会用自适应编码，就是预测方式会根据局部图像内容变化。比如：平滑区域：用周围像素平均值预测、边缘区域：更相信边缘方向上的邻居、纹理区域：换另一种预测模式，它不是全图只用一个固定公式。
有的人会问：预测值是依靠周围的像素的出来的，一个点需要预测可以这样做，但是如果一张图里面的每个点都需要预测，那么到底用谁来预测？你也并没有预存关键像素啊？
我们通常在处理边界条件的时候，比如第一个像素我们会特殊处理，比如他的真实值103，默认预测值是0，我们就记住误差值是103，剩下的就依靠已经处理完的、编码完了的相邻像素值对他进行处理。左边缘的像素可以只用上边预测，上边的像素用左边的像素预测。编码端：已经知道左边/上边原始像素、用它们预测当前像素、存误差；解码端：已经恢复出左边/上边像素、用同样公式预测当前像素、读取误差、预测值 + 误差 = 当前像素。因为左边/上边已经完全恢复，所以预测值和编码端一模一样。

一个例子：原始一行像素：100, 102, 103, 105
预测规则：第一个直接存，后面每个像素用左边像素预测
编码：p1 = 100，直接存 100
p2 预测值 = 100，误差 = 102 - 100 = 2
p3 预测值 = 102，误差 = 103 - 102 = 1
p4 预测值 = 103，误差 = 105 - 103 = 2
存的是：100, 2, 1, 2
解码：恢复 p1 = 100
p2 = 左边100 + 误差2 = 102
p3 = 左边102 + 误差1 = 103
p4 = 左边103 + 误差2 = 105
完全恢复。

我们需要明白的是压缩的最终目标是压得要小，看起来要好，角度一致性要好，光场一致性不要被破坏，再就是计算的复杂度和功耗也要合理。有的人可能会问，既然有损压缩和无损压缩都是用预测值+误差，那么这两个的区别在哪里？其实这两个的核心思路确实是这样的，但是他们的差别主要在于对于误差数据的处理不同。有损压缩对误差数据是存一个大概或者直接不存，但是无损压缩对误差数据就是完完整整的进行存储。

VII.光场应用
这一讲讲了光场除了拍照、重聚焦等等之外还能完成哪些视觉任务。

第一个，处理遮挡物体。
首先论文提出了合成孔径成像的方法，意思就是用很多小相机/很多视角来合成出一个完整的新的大孔径图像。比如我们要看清楚一个东西，但是这个东西被树叶或者什么铁丝网、栅栏之类的东西挡着，我们就用这个。核心思路是：对于单个视角来看这个东西确实被遮挡着，但是在不同的视角下这个东西总是会露出来一部分，所以不同的视角能看到这个物体的一部分，那我们把这些视角进行对齐融合不就能恢复这个东西的全貌了吗？他的方法也很简单，假设我们想要看到一个背景目标，这个目标的深度在z，那么我们按照这个深度把所有视角进行对齐，这个时候目标层对齐，但是剩余的比如遮挡层并没有对齐，再把所有视角进行平均/模糊，因为目标层对齐了，所以他不会变模糊，但是由于剩余层并没有对齐，那么就会被平均模糊。

在论文里面还提及了另外几种方法：第一种是利用代价函数来进行，论文提出了4种代价函数：color medians；entropy；focus；stereo，其核心思路都是先让目标物按照一定的深度对齐、如果深度对了目标物是齐的但是其余遮挡物是模糊的、用代价函数来判断对齐质量、选出一个代价函数最好的深度，但是这四个代价函数都有区别：stereo主要是看颜色有没有对齐，如果对齐了，那么从各个视角来看颜色都是一致的，所以看颜色的方差小不小，但是这个代价函数在遮挡物混进来的时候颜色一致性会被破坏；focus主要看边缘，如果深度对，对齐后目标表面清楚，边缘锐利，所以看重聚焦图像是否清晰，但是在表面没纹理时，清晰度不好判断；color median主要看大部分颜色是不是目标色，这里不用平均颜色，而用中位数颜色，看中位数颜色是否是目标物颜色，如果是那么就是好的，但是中位数对少量遮挡 outlier 更稳，如果遮挡太严重，超过一半视角都看到遮挡物，中位数也会偏向遮挡物；entropy主要看颜色分布乱不乱，如果深度对，来自真实表面的颜色集中，熵低；如果混了遮挡物，颜色分布更乱，熵高，但是问题在于严重遮挡时也会失败，所以论文评价这个方法在遮挡过于严重的情况下无法达到有效目标。

第二种方法就是多个可见性层。意思就是把一张图像分成了好几个可见性层，比如前景层、中间层、背景层等等，我们在这些层里面看到的是从某个相机/某个视角的某个光线看过去这个位置能不能看到，还是被前面的某个物体挡住了？所以我们就可以知道这些光线中哪些光线对背景层可见哪些不可见，所以可见背景层的光线就可以被用来恢复图像，不可见的光线就没有办法来恢复。这样做有一个好处就是相比之前的只有一层深度是清晰的，用可见的光线恢复能使得更大深度范围内都能保持比较清晰的状态。

第三种就是体积重聚焦，他是能够选定一段深度范围让它变得更加清楚，并且能够抑制噪声。

论文还提出合成孔径成像不仅能够只是单纯的恢复图像的目标，还能够进行在遮挡情况下的目标跟踪。普通的目标跟踪一般会采取卡尔曼滤波以及粒子滤波。卡尔曼滤波是通过这一帧来预测下一帧位置，而粒子滤波是用很多候选位置粒子表示目标可能在哪里，再根据观测更新概率，适合更复杂运动。但是如果目标被遮挡物遮挡，这些方法会失效。但是光场相机可以通过合成孔径使得遮挡物被虚化，让目标更加清晰，能够更稳定的跟踪目标。


第二个应用，分割和抠图。
首先是分割。
论文说，分割是一种标签问题。意思就是分割其实就是给图像贴标签，有些像素要贴可能是人的标签，有些要贴天空，有些要贴树......在光场中给像素贴标签的难点在于首先他要给全部的光场进行处理，其次就是给光场像素贴标签还要保证各个视角之间像素贴的标签一致，不能左边贴的是人结果到了右边贴的是树。但是光场的优点在于光场图像比普通的图像多了EPI、深度等等信息，可以很容易的看出比如前景和背景颜色相近时候的情况。我们在给像素贴标签的过程其实就是让每一次贴的标签更加的合理，能更明白的体现图像的情况。这个过程遵守能量最小化的定律，也就是E = 数据项 + 平滑项 + 一致性项，数据项就是指这个像素像不像前景/背景，平滑项是指相邻像素的标签是否应该相似，一致性项是说不同视角中对应的光线/点，标签是否一致？我们之前说的让标签更合理就是在贴标签的过程中要寻找一组标签，让这个能量最小。

论文对于让能量最小化提出了一种方法：图割。他的思想是将图像的像素进行分割，看怎么样把像素切的更合理代价更小。论文说有人尝试把像素按照图像空间以及体素空间进行分割，按照图像控件进行分割意思就是在每一张2D图像上进行像素分割，体素空间分割就是在3D空间中把像素分成一个一个的小格子，然后对这些小格子像素进行划分切割。但是论文说这种方法（按照周围像素的标签情况来判断这一个像素的位置和情况，又称MRF框架）因为光场中视角太多所以太过耗时。
为了解决这个问题，论文说有人利用EPI结构将这种分割问题变成了一种能量最小化的方案。什么是EPI结构？我们知道什么是EPI，就是选一排相机，把这一排相机的所有照片按照某一行像素看变化。那EPI结构就是把EPI按照顺序堆叠起来的一个立体结构，说的更明白一点，我们这个时候不再固定选某一排相机，我们把所有排相机全都选上，然后按照相机的排数这个规律把EPI堆叠起来就形成了EPI空间。由于EPI里面的线包含了空间-角度信息，所以我们在光场分割的时候利用EPI 里的线在同一物体点形成连续线、不同深度斜率不同、边界处线会断裂或交叉等等情况把不同深度、不同情况的物体分清。
论文还提及，有人对这种问题提出了变分框架。意思就是他们认为图像分割的结果应该是一个连续函数，然后最小化某个能量让不仅符合数据又平滑又保留边界。这个和像素分割的区别在于像素分割是离散化的贴标签，但是变分框架是连续函数的优化方式。
论文还说，有人根据手持摄像机拍摄到的一系列不规则、不整齐的多视角图像，通过利用梯度（梯度是用来判断某个地方变化大不大的，比如对于平坦区域他的梯度就比较小，相反的在边缘地方的梯度就比较大）以及梯度滤波器对某个边缘是否属于前景的概率进行了估计，并且使用贝叶斯对概率进行了整合，用以更加稳定的判断某个边缘是否属于前景的可能性。
论文又提出，有人用了这样一种方法：在子视角图像中把相邻的、颜色相近的像素合并成了一个稍微大一点的区域或者像素，称之为超像素，然后根据深度图我们就可以知道这些地方在别的视角图里面出于什么位置，我们就能把这个区域的标签传递过去。这种方式能将原来非常多的像素变成更少一点的像素，提升了运行的速度。
当然了，现实情况中总会出现透明的东西，比如玻璃杯之类的。这种东西要处理起来比较困难，因为光线在这些东西附近发生了扭曲，比如玻璃杯会扭曲后面的背景，而不是单纯的遮挡关系。那么对于这类东西我们会用视差、一致性、颜色对比度等等来对透明物体进行处理。

其次我们再来谈谈抠图。抠图的过程是一个比分割更加精细的过程，因为分割只是确定这个像素属于什么标签，但是抠图不仅要知道这个像素是什么标签，还要知道这个像素里面混杂了多少比例的前景多少比例的后景。有人会有疑惑，说，我抠图抠图只需要知道这个像素是什么东西然后把这个像素抠出来不就好了吗？为什么要搞这么细致呢？因为如果按照只是知道这个像素属于什么标签然后按照这个标签抠下来的话，有些时候会显得非常假。当然我们说如果是一张白纸上面的有一个黑色方块，那么这种方法是可以的。但是实际情况中比这个要复杂许多，因为相机的像素不能无穷小，所以这就导致了有些像素会混杂别的部分。比如头发丝，头发丝的像素可能会混杂一部分背景，如果只是按照像素标签进行抠图的话，那么抠出来的头发丝必定会有一部分背景，就会显得很假。所以我们在抠图的过程中不能仅仅是按照像素的标签进行判断，我们要计算像素的透明度，也就是我们要估计这个像素里面有多少比例是属于前景的，这就是文章里面所提到的alpha。比如一个边缘像素：alpha = 1，说明完全是前景；alpha = 0，说明完全是背景；alpha = 0.3，说明这个像素大概 30% 是前景，70% 是背景。这种方式能够很好的让我们把抠出来的图放在一个新背景里面，如果我们只是按照像素标签进行抠图的话，把抠出来的图放在新背景里面必定会产生与背景不符的情况，如果有了alpha图，也就是论文所提及的alpha matte，我们就能通过公式（新图像 = alpha × 前景 + (1 - alpha) × 新背景）将抠出来的图很好的融合进新背景里面。那么我们怎么得到这个alpha图呢？首先我们根据像素标签得到一个大致的区域，然后我们会得到一个三值图trimap，分成了前景部分、后景部分以及不确定区域，之后我们只是在不确定区域计算alpha，得到更加精细的alpha matte。这就是抠图的经典流程。


第三个应用，图像物体的检测、分类以及识别
首先我们区别一下这三个词汇的概念，检测指的是算法在图像里面找有没有某一类物体，如果有他在哪里，所以输出的是位置，比如输入一张图片，这个算法就要知道这个图片里面有没有猫，如果有猫猫在哪里；分类指的是要知道输入的图像算是哪一类，比如输入的图片有猫有狗，我们就要对图像进行分类，知道哪些图像算是猫，哪些图像算是狗；识别要更加细致一些，他不仅要认出这是什么东西，还要知道这是谁的，位置在哪里，比如输入一张图片，算法就要知道这个图片里面有没有猫，有猫的话是什么种类的猫，在哪里，或者说要认出来这是一张人脸，然后还要认出来这是谁的脸，是张三的脸还是李四的脸。

在辨明概念之后，我们正式开始。

论文说光场图像可以在显著性检测上面发挥很大的作用，因为光场可以利用重聚焦的功能对图像的背景和后景进行似然度评分，这样能够更好的处理前景和后景混杂的复杂问题。这一句话是这个意思：首先我们来解释什么是显著性检测，显著性检测就是找出图像内部最吸引注意力的那个区域，比如哪个地方是前景的主体，普通图像的算法进行显著性检测的时候通常会依赖边缘、颜色等等，但是光场图像因为其特殊性，所以可以利用重聚焦把前景或者后景进行模糊处理，然后对每个部分是否是前景或者后景计算概率，这就是似然度评分。
论文还提出，有人利用多视角的图像对某一个深度进行融合对齐，能够实现多目标的检测。这是因为多视角图像对某一个深度进行融合对齐之后，如果目标在这个深度，那他就会变得清晰，相应的其他深度就会变模糊，所以在这个深度上更加容易检测目标。在这个思路的基础上，论文说有人提出了最佳聚焦图这个概念，意思就是从一组重聚焦图中选出最清楚的那一张，或者把每个区域最清楚的部分组合起来，比如做人脸检测：先用光场重聚焦、找到脸最清楚的焦平面、再在这张图上做检测。那我们该如何找出这个最佳聚焦图呢？论文说我们可以用小波能量检测法进行分析。什么意思？小波是一种尺度分析工具，可以检测不同尺度的细节，那么小波能量就是图像在某个尺度上的细节强度，清楚的图像通常边缘和纹理更强，小波能量更集中，所以可以用它判断哪张重聚焦图最清楚。
除了对正常物体的识别，对材料的识别也是一件比较麻烦的事情，因为对于某些材料比如金属，他的高光从不同的角度看都不一样，这就导致光线不同，但是光场图像由于记录了光线的变化，所以对于金属的识别相对轻松一些。论文说有人使用CNN对4D的数据集进行识别，主要是学习不同材质随视角变化的特征。

第四个应用，处理折射和透明物体
由于水面、玻璃等物体会对光线产生折射现象，并且由于透明物体颜色本身较浅、边界不明显，所以难以被识别。论文说，有人根据这些物体对光线的扭曲特性，认为可以检测哪里发生了光线扭曲，哪里就可能有透明物体。论文还提出，针对于流体以及火焰的重建，有人提出了平移不变重聚焦变换法，其核心思想是一些表面物体可以根据表面深度进行描述，但是对于流体和火焰这种体积现象，我们需要对体积深度的光学变化进行估计。什么是平移不变重聚焦法？拆开来看：平移不变就是某种处理对位置平移不敏感，或者可以用统一方式处理不同位置；重聚焦变换就是通过移动/积分不同视角来聚焦某个深度。还有人用时空对应关系图来进行流体表面的恢复和重建，意思就是这个时刻/这个相机看到的点，对应另一个时刻/另一个相机中的哪个点，这种跨相机跨时间对特征点进行跟踪的思路主要是希望通过流体表面光线或者背景光线的弯曲程度来反推流体表面的形状。
对于会产生反射光的物体（比如陶瓷杯），有人采取将漫反射和镜面反射分离的光场技术。他们认为我们观察到的颜色实际上是一部分颜色恒定的镜面反射项和一部分物体固有颜色的线性组合，换句话说，他们认为他们认为观测到的光场颜色，可以由两部分组成：一部分是镜面高光，颜色近似固定；另一部分是少数几种物体固有颜色的组合，而且每个局部位置真正用到的成分很少。举个例子，我现在有一个有高光的红色的陶瓷杯子，某个像素看到的颜色可能是：红色杯子本身的颜色 + 白色高光，模型就写成：观测颜色 ≈ a * 白色高光项 + b * 红色，如果场景里还有蓝色图案，可能是：观测颜色 ≈ a * 白色高光项 + b * 红色 + c * 蓝色，但 b,c 不会所有颜色都用上，只用少数几个。由于光场记录了不同视角下高光的变化，所以使用光场技术对将高光和固有颜色进行分离较为容易。有人从不同材料特质对光线的反射特性入手，建立了BRDF也就是双向反射分布函数，这个函数旨在描述光从某个方向照到表面后，会向各个观察方向反射多少，也就是材质如何反光的函数。比如：哑光材料各方向差不多、镜面材料某些方向特别亮、金属高光强且颜色特殊、布料反射比较散，通过CNN对这些不同特质材料的反光特性进行学习，用以识别不同的材料。

第五个应用，应用在机器人以及三维视觉。主要分为：视觉里程计、定位以及运动恢复结构。
论文首先说有人提出了6个自由度（平移：x, y, z；旋转：roll, pitch, yaw）的视觉里程计用来估计光场的全光流（光场下的光线样本随着时间或者相机运动如何变化），这样比传统的双目特征跟踪（在左右相机内部同时找一个同样的特征点，再在时间上匹配这两个点）更加稳定，因为双目的特征跟踪更多的是依赖特征点的匹配，而光场利用的是更多的视角，是一种动态的连续跟踪，所以视觉里程计比双目特征跟踪更加稳定。
其次有人在全传感器的高分辨率图像中寻找不同视角的对应关系（比如左边视角的这个点对应右边视角的哪个点）。为什么他不在已经解码好了的子视角图中找对应关系呢？因为原本的图像是高分辨率的，但是单独抽出来的子视角图会是低分辨率的，如果在低分辨率的子视角图中进行双目特征匹配，会因为子视角图的空间欠采样导致混叠，也就是边缘模糊、匹配点不准等一系列问题。
再然后论文说有人提出了有人采用带颜色编码的背景板的透镜，让不同方向来的光对应不同颜色，这样我们可以根据观察到的颜色来反推出光线的方向，从而简化相机的旋转估计。意思就是我们用不同的颜色对不同方向的光线进行了编码，因此我们的相机在转动中能看到不同的颜色编码，只要观察颜色编码的变化我们就能知道相机转到了哪个位置。
然后论文还说有人在4D的光场上面取了一个倾斜的局部窗口，用于精准的像素对比来恢复场景流。为什么是倾斜的窗口？因为同一个 3D 点在光场里不是垂直堆叠，而是沿视差方向移动。如果窗口方向和视差一致，比较像素时就更准确。有人会问为什么是开了一个局部的窗口？因为如果进行单独像素的比对，可能会因为像素和像素之间过于相似比对失败，但是如果我们把窗口放大，我们就能得到更加多样化的信息，对比更加准确。
最终论文说，有人还根据3D流的运动对速度场进行准确的估计。意思是我们既然已经知道了不同时间（比如时间t以及时间t+1）的光场图像，我们就可以通过这个来估计不同场景中每个3D块如何运动，每个3D块的位移除以时间就能得到速度，于是我们得到了速度场。

第六个应用，也就是在视频方面的应用，分为多图对齐、去模糊以及去抖动。
首先我们来讲光场图像的对齐。对于普通的2D图像，我们在进行图像对齐的时候需要进行点对点对齐，而在光场图像对齐的时候我们进行的是光线到光线的对齐过程，所以我们不仅图像边缘要对齐，光线方向和多视角结构也要对齐。所以有人提出了光线空间运动矩阵，希望能通过这个矩阵把一个光场里面的光线坐标变换到另外一个光场里面的矩阵。为什么这么说？是因为不同的视角所采用的坐标可能不同，我们要把不同视角的图像进行对齐，就需要知道某一个视角的某一条光线在另外视角里面是哪一条、位置在哪里，才能知道如何把光线进行对齐，所以需要进行光线的坐标变换。
其次就是3D去模糊。对于普通的2D视频，我们在去模糊的时候用到的是假设模糊图像=清晰图像和模糊核进行卷积（模糊核是描述这个2D像素被拖成什么样子的）。但是对于3D图像，由于不同深度的物体模糊程度不一样，所以我们不能用2D的方法进行去模糊。所以有必要建立一个3D相机的运动模型来进行去模糊。
最后就是视频稳定。普通抖动视频稳定：估计相机抖动、裁剪/变换图像、生成平滑相机路径。光场/相机阵列更强，因为它可以在已有多视角中合成虚拟相机视角。也就是说，不只是把画面平移裁剪，而是可以生成一个更平滑轨迹上的虚拟相机视频。论文中提到有人通过时空同时优化（也就是同时考虑：空间上：图像/视角几何是否对齐；时间上：相邻帧运动是否平滑）让视频整体看起来更加稳定更连续，并且在视频稳定中估计出每一帧的相对位姿，就能知道相机怎么抖了。因此我们就能够实现3D视频的稳定。
